# 🗺️ Roadmap — Eranova Docs

This roadmap outlines the planned evolution of the Eranova Workforge documentation. Priorities may shift based on community feedback and platform development.

---

## 🟢 Phase 1 — Foundation (Current)

> Establish core documentation infrastructure

- [x] Docusaurus 3 setup with TypeScript
- [x] Project scaffolding and CI/CD
- [x] Contributing guide and community standards
- [x] Basic tutorial documentation
- [ ] Custom branding and design system
- [ ] SEO and Open Graph metadata
- [ ] Deployment to production domain

## 🟡 Phase 2 — Core Documentation

> Cover the essential platform features

- [ ] **Getting Started** — Onboarding guide for new users
- [ ] **Platform Overview** — Architecture and key concepts
- [ ] **Talent Discovery** — How to find and match with opportunities
- [ ] **Network & Connections** — Growing your professional graph
- [ ] **Funding Tools** — Capital raising, investor matching, grants
- [ ] **Team Building** — Procurement workflows and team management
- [ ] **API Reference** — Auto-generated from OpenAPI specs

## 🔵 Phase 3 — Developer Experience

> Make it easy for developers to integrate and extend

- [ ] REST API documentation with interactive examples
- [ ] SDK guides (JavaScript, Python)
- [ ] Webhook integration guides
- [ ] OAuth 2.0 / authentication flows
- [ ] Rate limiting and best practices
- [ ] Code samples repository
- [ ] Postman/Insomnia collection

## 🟣 Phase 4 — Advanced & Community

> Scale the documentation with community involvement

- [ ] Versioned documentation (multi-version support)
- [ ] Internationalization (i18n) — starting with Spanish, French
- [ ] Video tutorials and walkthroughs
- [ ] Community-contributed guides
- [ ] Case studies and success stories
- [ ] Architecture Decision Records (ADRs)
- [ ] Plugin/extension development guide

## 🔮 Future Ideas

- AI-powered documentation search
- Interactive API playground
- Embedded code sandboxes (StackBlitz)
- Documentation analytics dashboard
- Auto-generated changelog from commits
